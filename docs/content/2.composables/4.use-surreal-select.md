# useSurrealSelect

Select records from a SurrealDB table with a builder-pattern API and automatic SSR support.

## Basic usage

```vue [app/pages/index.vue]
<script setup lang="ts">
const { data } = await useSurrealSelect(new Table('users'))
</script>
```

## With query builder

The composable accepts an optional builder callback for filtering, pagination, and field selection:

```vue [app/pages/index.vue]
<script setup lang="ts">
const { data } = await useSurrealSelect(
  new Table('users'),
  (select) => select
    .fields('name', 'email')
    .where(eq('active', true))
    .limit(10)
    .start(0),
)
</script>
```

::tip
Expression helpers like `eq`, `gt`, `contains`, etc. are auto-imported when `autoImportExpressions` is enabled in your config.
::

## Select a single record

```vue
<script setup lang="ts">
const { data } = await useSurrealSelect(new RecordId('users', 'john'))
</script>
```

## Builder methods

| Method | Description |
|--------|-------------|
| `.fields(...fields)` | Select specific fields |
| `.value(field)` | Retrieve the value of a single field |
| `.where(expr)` | Filter records using an expression |
| `.limit(n)` | Limit the number of results |
| `.start(n)` | Skip the first `n` results |
| `.fetch(...fields)` | Fetch record link contents |
| `.timeout(duration)` | Set query timeout |
| `.version(dateTime)` | Query a specific version (SurrealKV) |

## Arguments

| Argument | Type | Description |
|----------|------|-------------|
| `tableOrRecord` | `Table \| RecordId \| RecordIdRange` | The target table or record |
| `select?` | `(select) => select` | Builder callback |
| `asyncDataOptions?` | `AsyncDataOptions & { key?: string }` | Options passed to `useAsyncData` |

## Return type

Returns the same shape as [`useSurrealAsyncData`](/composables/use-surreal-async-data).
